## Project Overview

**Cortex** is an Obsidian plugin that brings Claude Code's agentic file management capabilities into the Obsidian environment. The full vault serves as the project context, with Claude able to read, write, create, move, and organize notes — while the user retains precise control over what Claude can see, touch, and remember.

**Key design constraint:** Cortex uses the Claude Code CLI binary as a subprocess — it does NOT use the Anthropic Messages API directly and does NOT use the Claude Agent SDK. This approach requires no API key; it rides the user's existing Claude Pro/Max subscription, the same way Cline and Zed work.

## Architecture (Core Decisions — Do Not Revisit Without Discussion)

### Integration approach
- Spawns `claude` CLI binary as a child process (`child_process.spawn`)
- Uses `--output-format stream-json --print` for non-interactive use
- No Anthropic API key required; uses Pro/Max subscription auth
- Vault root is the `cwd` for all claude process spawns

### Session storage
- All sessions stored inside the vault under `.obsidian/claude/sessions/`
- Session files are JSON (see format in design doc)
- `.obsidian/` is typically gitignored — sessions won't pollute version control

### Context system (replaces CLAUDE.md)
- No `CLAUDE.md` used at the vault level — plugin manages its own context
- Context file default path: `_claude-context.md` at vault root (user-configurable)
- Four context layers: vault tree, context file, pinned notes, inline selection
- Per-note frontmatter schema under `claude:` key controls access and behavior

### Frontmatter schema
```yaml
claude:
  readonly: true       # blocks writes
  protect: true        # alias for readonly
  context: always      # permanently pinned to every session
  context: never       # excluded from Claude's access entirely
  instructions: "..."  # injected when this note is read or referenced
```

## Build Stack

- **Language:** TypeScript
- **Bundler:** esbuild (standard for Obsidian plugins)
- **Runtime deps:** None (everything bundled)
- **Dev deps:** `@types/node`, Obsidian API types
- **Node APIs used:** `child_process`, `path`, `os`, `fs`
- **Desktop only:** `isDesktopOnly: true` in manifest — no mobile support

## Development Workflow

### Prerequisites (user's machine)
- Node.js 18+
- npm
- Claude Code CLI installed and authenticated **in the native Windows environment (PowerShell), not only in WSL**
  - The plugin spawns `claude` as a Windows child process — WSL-only installs are invisible to it
  - Install: `irm https://claude.ai/install.ps1 | iex` (run in PowerShell), or `winget install Anthropic.ClaudeCode`
  - Verify: `claude --version` must work in a plain PowerShell window
- Obsidian desktop
- A throwaway test vault (don't develop against real vault until stable)

### Build commands (once scaffolded)
```bash
npm install       # install deps
npm run build     # one-shot build → main.js
npm run dev       # watch mode (use with Hot Reload plugin in test vault)
```

### Test vault setup
```bash
# Mac/Linux
ln -s /path/to/cortex /path/to/test-vault/.obsidian/plugins/cortex

# Windows (PowerShell as admin)
New-Item -ItemType SymbolicLink -Path "C:\vault\.obsidian\plugins\cortex" -Target "C:\dev\cortex"
```

Reload plugin: Ctrl/Cmd+P → "Reload app without saving" (or Hot Reload plugin)

## Version Control & Release Process

- **Commit style:** [Conventional Commits](https://www.conventionalcommits.org/)
  - `feat:` new feature
  - `fix:` bug fix
  - `docs:` documentation only
  - `chore:` maintenance
  - `refactor:` code change with no behavior change
  - `test:` tests
  - Breaking changes: add `!` after type or `BREAKING CHANGE:` in footer
- **Release automation:** [release-please](https://github.com/googleapis/release-please) generates `CHANGELOG.md` and version bumps from commit messages
- **Branching:** `main` is the primary branch; PRs for all changes
- **Do not:** amend published commits, force-push to main
